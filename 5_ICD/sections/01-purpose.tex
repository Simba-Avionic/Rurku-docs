\section{Purpose of this document}

An ICD records, defines, and controls interface information between systems,
subsystems, equipment, hardware, or software. It is the shared reference when
separately developed sides of a connection must stay compatible --- signals,
protocols, fittings, limits, and responsibilities.

Without an ICD, each team invents pinouts and fittings; integration fails late.
With an ICD, changes go through impact analysis on \emph{both} sides before they
become ``approved''.

\noindent\textbf{Audience:} owners of both sides of each \texttt{IF-*}; anyone
integrating vehicle and GSE; anyone writing interface tests in TPTR.

\noindent\textbf{Out of scope:} subsystem tree and budgets (SAD); Must/Should
tables (SRD); pad timelines and abort scripts (CONOPS); test evidence (TPTR).
