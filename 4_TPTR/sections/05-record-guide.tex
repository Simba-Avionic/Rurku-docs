\section{Record structure (what each test file contains)}
\label{sec:record-guide}

Each \texttt{records/TP-*.tex} file is one test. It always has two parts.

\subsection{Plan (before)}

\begin{itemize}
  \item Objective --- why this test exists
  \item Configuration --- hardware / software / site
  \item Procedure --- steps (or pointer to CONOPS checklist)
  \item Success / failure criteria --- measurable pass/fail
  \item SRD IDs verified --- which requirements this closes
  \item Hazards / safing --- especially for fueled / armed tests
\end{itemize}

\subsection{Report (after)}

\begin{itemize}
  \item Results --- numbers, pass/fail vs.\ criteria
  \item Plots / data --- paths or embedded figures
  \item Photos --- what to keep in the archive
  \item Anomalies --- what went wrong or unexpected
  \item Conclusions --- change design / CONOPS / retest / proceed
\end{itemize}
